# AgentFence Roadmap

## Stage 1: Public CLI

Goal:

Make the scanner useful enough for developers to run in local projects and share reports.

Deliverables:

- Local config discovery.
- Risk rules for MCP and agent configs.
- JSON and HTML reports.
- Fixtures and tests.
- GitHub Action CI.

Status: in progress.

## Stage 2: Distribution

Goal:

Earn attention from developers and AI-agent builders.

Deliverables:

- GitHub repository.
- Public README with screenshots.
- npm package.
- Demo video or GIF.
- Launch posts for Hacker News, Reddit, LinkedIn, X, and MCP communities.

Validation targets:

- 100 GitHub stars.
- 500 npm downloads.
- 20 waitlist signups.

## Stage 3: Pro Features

Goal:

Convert active usage into paid workflow value.

Deliverables:

- GitHub Action policy gate.
- `agentfence.yml` policy file.
- Approved tool registry.
- PR comments.
- SARIF output.
- VS Code/Cursor extension.

## Stage 4: SaaS Dashboard

Goal:

Turn one-off scans into team risk monitoring.

Deliverables:

- Project dashboard.
- Historical scan trends.
- Team policy management.
- Slack/email alerts.
- Paid plans.
