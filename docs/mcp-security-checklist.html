MCP security checklist

Review MCP permissions before agents touch real systems.

MCP servers can give agents access to files, shell commands, APIs, databases, and internal tools. This checklist keeps the first review simple. 

npx agentfence scan --path . \
  --sarif agentfence-report.sarif \
  --fail-on high

Checklist

Minimum review before using MCP in a project.

Scope filesystem access

Do not grant home, drive, root, system, or broad workspace access unless the workflow truly needs it.

Keep secrets out of configs

Move API keys, tokens, and passwords into a secret manager or environment controlled outside agent-readable files.

Block destructive commands

Require human approval before tools can delete files, drop databases, destroy infra, or modify production systems.

Use least privilege

Replace wildcard, admin, all, shell, network, and write scopes with the smallest useful permission set.

Treat external text as hostile

Tool descriptions, retrieved documents, web pages, and issue text can contain prompt-injection instructions.

Run scans in CI

Add a CI check so risky agent permission changes are reviewed before merging.
